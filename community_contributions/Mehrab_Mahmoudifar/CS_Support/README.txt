راهنمای جامع پروژه‌ی نهایی مهندسی هوش مصنوعی عامل‌محور

این مستند شامل دستورالعمل‌های گام‌به‌گام جهت راه‌اندازی و اجرای سیستم هوشمند خدمات مشتریان است. این پروژه به عنوان بخشی از دوره‌ی مهندسی مدل‌های زبانی بزرگ (LLM Engineering) طراحی شده و از فریم‌ورک‌های پیشرفته‌ای نظیر LangGraph برای مدیریت وضعیت و Streamlit برای رابط کاربری بهره می‌برد.

معرفی پروژه
این سیستم یک دستیار هوشمند فروشگاهی است که با استفاده از تکنیک تولید مبتنی بر بازیابی (RAG)، اطلاعات محصولات و سوالات متداول را از پایگاه داده‌های محلی استخراج کرده و به کاربران پاسخ می‌دهد. از ویژگی‌های برجسته این پروژه می‌توان به تشخیص هوشمند نیت کاربر برای ثبت سفارش و ذخیره‌سازی داده‌ها در فایل‌های محلی اشاره کرد.

پیش‌نیازهای سیستم
برای اجرای بدون نقص این پروژه، موارد زیر الزامی است:

    پایتون نسخه 3.11 یا بالاتر (تست شده بر روی نسخه 3.13.5)

    دسترسی به اینترنت جهت ارتباط با API سرویس AvalAI

    کلید اختصاصی API از پنل کاربری AvalAI

مراحل راه‌اندازی و اجرا

۱. آماده‌سازی محیط محلی
ابتدا مخزن پروژه را در سیستم خود کلون کرده و وارد پوشه اصلی پروژه شوید. سپس یک محیط مجازی (Virtual Environment) برای جداسازی وابستگی‌های پروژه ایجاد کنید:

python -m venv venv

۲. فعال‌سازی محیط مجازی
بسته به سیستم‌عامل خود، محیط مجازی را فعال کنید:

    در ویندوز:
    venv\Scripts\activate

    در مک یا لینوکس:
    source venv/bin/activate

۳. نصب وابستگی‌ها
تمام کتابخانه‌های مورد نیاز در فایل requirements.txt درج شده‌اند. با دستور زیر آن‌ها را نصب نمایید:

pip install -r requirements.txt

۴. تنظیمات متغیرهای محیطی
یک فایل با نام .env در ریشه اصلی پروژه بسازید و کلید API خود را طبق فرمت زیر در آن قرار دهید:

AVALAI_API_KEY=کلید_اختصاصی_شما
AVALAI_BASE_URL=https://api.avalai.ir/v1

۵. پایگاه داده‌های محلی
اطمینان حاصل کنید که فایل‌های products.json و faq.json در کنار فایل اصلی برنامه قرار دارند. این فایل‌ها شامل اطلاعات محصولات و پاسخ سوالات متداول هستند که ایجنت برای پاسخگویی به آن‌ها تکیه می‌کند.

۶. اجرای برنامه
برای اجرای رابط کاربری گرافیکی، از دستور زیر استفاده کنید:

streamlit run main.py

پس از اجرای این دستور، یک آدرس محلی (معمولاً http://localhost:8501) در ترمینال نمایش داده می‌شود که با باز کردن آن در مرورگر، می‌توانید با دستیار هوشمند گفتگو کنید.

ساختار فایل‌های پروژه

    main.py: کد اصلی برنامه شامل گراف‌های تصمیم‌گیری و رابط کاربری.

    final_project.ipynb: نسخه نوت‌بوک جهت تست گام‌به‌گام توابع و خروجی‌ها.

    products.json: پایگاه داده محصولات فروشگاه.

    faq.json: پایگاه داده سوالات متداول.

    orders.json: محل ذخیره‌سازی سفارشات ثبت شده توسط کاربران (پس از اولین سفارش ایجاد می‌شود).

    requirements.txt: لیست کتابخانه‌های مورد نیاز.

نکات فنی

    این پروژه از ساختار StateGraph برای مدیریت حافظه کوتاه‌مدت و بلندمدت گفتگو استفاده می‌کند.

    در صورت بروز خطای محدودیت درخواست (Rate Limit)، سیستم به طور خودکار پس از چند ثانیه تلاش مجدد خواهد کرد.

    برای مشاهده سفارشات ثبت شده، می‌توانید از پنل مدیریتی موجود در نوار کناری (Sidebar) برنامه استفاده کنید.
